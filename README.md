# learning skill

一个轻量的 Codex 项目学习模式 skill。

它不是完整课程系统，也不替代 `teach`。它做的事情更窄：在普通任务推进过程中，于自然阶段边界补充解释“为什么这么做”，让使用者顺手理解判断依据，但不影响任务质量。

## 使用方式

显式调用 `learning` 用来切换当前项目的学习模式：

```text
/learning
```

第一次显式调用会在当前项目创建：

```text
.codex/learning/
├── NOTES.md
├── OMISSIONS.md
└── state.json
```

并把 `state.json` 设为：

```json
{
  "enabled": 1
}
```

之后每次显式调用都会翻转状态：

```text
1 -> 0
0 -> 1
```

开启时，脚本会在项目 `AGENTS.md` 中写入一个 `learning-managed` 托管块，让新对话知道要读取全局 `learning` skill 并做只读状态检查。关闭时，脚本会移除这个托管块；如果 `AGENTS.md` 只剩这个托管块，则删除该文件。

显式调用只是开关。切换到开启后，当前线程应加载学习模式规则和项目 `NOTES.md`；当前回复是否补充教学，由学习模式的阶段边界规则决定，而不是由“刚刚开启”这个动作决定。

## 运行行为

学习模式开启后，Codex 只在任务自然阶段边界补充简短解释，例如：

- 读完上下文后：解释为什么先看这些文件。
- 做关键设计选择前：解释取舍。
- 修改完成后：解释为什么这样改。
- 测试失败或通过时：解释信号说明了什么。

默认每次 1-3 句。任务质量优先，教学只是附带说明；它不能改变任务目标、扩大范围或降低验证质量。

## 项目记忆

`.codex/learning/NOTES.md` 只记录“以后怎么教我”的稳定偏好。

它不是状态文件、命令配置或制度规则。用户对教学不满意时，只有清楚、可复用的偏好才应该整理写入；前后矛盾时，重写“教学偏好”区域，让它变成自洽的偏好列表，而不是追加投诉流水。

`.codex/learning/OMISSIONS.md` 记录那些可能值得写入 `NOTES.md`、但暂时不够明确、不够稳定或不是教学偏好的候选项。它需要包含开始时间和简要背景，只作为审计线索，不是正式偏好。

`.codex/learning/` 是个人化项目记忆。脚本会在 Git 仓库中幂等维护 `.gitignore`，只忽略 `.codex/learning/`，不忽略整个 `.codex/`。

## 设计语境

### 核心词汇

**补充教学**：普通工作前后追加的一小段教学，用来解释关键判断和“为什么这么做”，但不把整个线程变成正式课程。

**项目状态**：每个项目自己的补充教学开关，用来决定这个项目是否允许 `learning` 执行补充教学。

**状态文件**：项目记忆目录里的 `state.json`，只保存 `enabled` 一个字段。它只承担开关门禁，不保存时间戳、教学偏好或历史记录。

**项目记忆目录**：当前项目下的 `.codex/learning/`，保存 `learning` 的项目状态和 `NOTES.md`。它是个人化代理记忆，默认应加入项目 `.gitignore`，不进入 Git 版本历史。

**显式调用**：用户直接点名 `learning` skill 的请求，例如 `/learning`、`$learning` 或 skill chip。显式调用用于创建或切换项目状态，影响接下来的对话是否进入学习模式。

**学习模式**：项目状态为 `enabled: 1` 后，后续普通对话进入的补充教学状态。学习模式开启时，agent 在自然阶段边界补充解释“为什么这么做”；学习模式关闭时不主动补充教学。

**项目接入规则**：`learning` 开启时由脚本写入当前项目 `AGENTS.md` 的固定托管规则块，用来让新对话读取全局 `learning` skill 并恢复学习模式；关闭时由脚本移除该托管规则块，同时保留 `AGENTS.md` 里的其他项目规则。项目接入规则触发的是只读检查，不是显式调用，不能改变项目状态。

**任务优先**：`learning` 不改变 agent 的任务执行标准。补充教学只能解释和压缩已有判断，不能驱动方案选择、扩大工作范围或降低验证质量。

**NOTES 职责**：`NOTES.md` 只记录“以后怎么教我”的稳定偏好，不能变成状态文件、命令配置或制度规则。

**遗漏记录**：`OMISSIONS.md` 记录“可能值得写入 NOTES.md，但因不够明确、不够稳定或不是教学偏好而未写入”的候选项。它只提供审计线索，不是正式教学偏好。

### 架构决策

我们决定把 `learning` 做成独立的全局 skill，但把开关状态和教学偏好放在各项目的 `.codex/learning/` 中，并在开启时由脚本向项目 `AGENTS.md` 写入一个固定托管接入块。

这个结构可以让当前对话通过显式调用随手切换学习模式，也能让新对话通过项目规则读取全局 `learning` skill 后恢复状态；同时 `.codex/learning/` 默认不进 Git，`AGENTS.md` 是否入库则保持项目现状。

这个结构比“只靠当前线程记忆”更可靠，也比“全局 AGENTS 永久扫描所有项目”更克制。项目 `AGENTS.md` 只负责接入，不复制 learning 行为、不硬编码本机绝对路径；具体是否启用和怎么教学仍由全局 `learning` skill、项目 `state.json` 和 `NOTES.md` 决定。
